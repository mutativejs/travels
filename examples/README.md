# Travels Examples

These examples focus on integration and product-shaped state, not counters.

| Example | What it demonstrates |
| --- | --- |
| [`basic.ts`](basic.ts) | Minimal vanilla usage |
| [`react-integration.tsx`](react-integration.tsx) | React subscription with `useSyncExternalStore` |
| [`zustand.ts`](zustand.ts) | Zustand as the UI store with Travels as the history engine |
| [`vue.ts`](vue.ts) | Vue composable wrapper |
| [`pinia.ts`](pinia.ts) | Pinia setup store with mutable mode |
| [`mobx.ts`](mobx.ts) | MobX observable state with mutable mode |
| [`form-builder.ts`](form-builder.ts) | Manual archive mode for multi-step form builder edits |
| [`canvas-editor.ts`](canvas-editor.ts) | Pointer-move batching for canvas/editor workflows |
| [`persistence.ts`](persistence.ts) | Versioned localStorage persistence |
| [`persistence-adapters.ts`](persistence-adapters.ts) | localStorage, IndexedDB, and compression adapter patterns |
| [`local-first-persistence.ts`](local-first-persistence.ts) | IndexedDB plus BroadcastChannel local-first persistence |
